# Tina 經驗文件庫 (Experience Ledger)

## 說明
這是 Tina 大腦的長期記憶庫。每次重大交易、策略修正或系統異常後，Tina 會將「因果關係」寫入此文件。
下次決策時，Tina 會自動比對過往經驗，避免重蹈覆轍。

## 2026-05-02 重大經驗記錄

### 經驗 #1
**日期**: 2026-05-02
**類別**: 策略修正
**情境**: RSI 進場區間過寬導致勝率下降

**觸發事件**:
- 系統在 RSI 65-70 區間進場 5 次，4 次虧損
- 原因分析：市場已進入過熱區，RSI 鈍化失效

**因果關係**:
- 當 RSI > 65 時，進場勝率僅 23%
- 修正後 RSI 進場區間改為 35-55

**教訓**:
- RSI 65 以上應禁止進場（適用所有股票）
- MA20 多頭排列掩護了 RSI 過熱的假訊號

**下次警示**: 當 RSI > 65 且無明確回調訊號時，不建議進場

**下次警示**: 當 TWII 歷史波動率 < 15% 且無明顯趨勢時，RSI 策略需搭配 MA 確認

### 經驗 #4
**日期**: 2026-05-02
**類別**: 產業輪動
**情境**: 金融股 vs 科技股的資金輪動

**觸發事件**:
- 2026-05 初，科技股 RSI 集體過熱（>80）
- 資金開始流向低基期金融股

**因果關係**:
- 金融股 PE 普遍 < 15，估值修復邏輯
- 當科技股 RSI > 75 時，預測資金將輪動至金融股

**教訓**:
- 每季檢視一次 RSI 分佈，識別過熱區塊
- 資金輪動預判：從高 RSI 流向低 RSI 區塊

**下次警示**: 當任一產業 RSI > 80 超過 5 個交易日，預測資金將輪動

### 經驗 #6
**日期**: 2026-05-02
**類別**: 腳本相容性修正
**情境**: 批量更新腳本在 Windows PowerShell 執行失敗

**觸發事件**:
- `batch_update_strategies.py` 包含中文字符和 emoji
- Windows cp950 編碼無法處理 Unicode 字符（如 ✅ ⚠️ 📝）
- 錯誤：`UnicodeEncodeError: 'cp950' codec can't encode character '\u2705'`

**因果關係**:
- PowerShell/cmd 環境預設使用 cp950（Big5繁體中文）
- emoji 和某些 Unicode 符號無法在 cp950 編碼顯示
- `sys.stdout.reconfigure(encoding='utf-8')` 無法完全解決問題

**教訓**:
- Windows 環境執行 Python 腳本時，避免在 print() 中使用 emoji/特殊符號
- 使用 ASCII 字符替代（如 [OK] [SKIP] [FAIL]）
- 或在 script 开头使用 `import io; sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')`

**下次警示**: 在 Windows 上執行含有中文輸出的腳本前，先確認編碼兼容性
