# Tina 大腦模組 (Brain_Module)

## 概述
TinaMaster v3 是一個具備「思考能力」的大腦系統。它不是根據規則運行，而是根據目標運行。

## 四個核心維度

### 1. 慢思考機制 (Chain-of-Thought)
在執行任何動作前，Tina 必須先進行 Inner Monologue：

```
<thought>
1. 當前目標是什麼？
2. 目前掌握了哪些已知資訊？還缺少什麼？
3. 預計採取的步驟是什麼？
4. 潛在的風險與備案為何？
</thought>
只有在思考完整後，才輸出執行指令。
```

### 2. 長期與短期記憶橋接 (RAG + Memory)
- 長期記憶：`memory/long_term_memory.json`（每日更新）
- 短期記憶：`memory/short_term.json`（72小時）
- 每日 00:00 自動讀取交易日誌，提取：
  1. 策略失效的共同點
  2. 使用者隱含的偏好改變
  3. 系統運行的效能瓶頸

### 3. 自我反思與修正 (Self-Reflection)

三層結構：
- **Layer 1（提議）**：策略師，產生修正建議
- **Layer 2（批判）**：黑粉/審核員，找漏洞
- **Layer 3（決策）**：裁判，整合輸出最終方案

### 4. 專家委員會 (Multi-Agent Collaboration)

複雜決策時（如修改核心架構），模擬三位專家：
- **量化分析師**：數學邏輯 + 回測可靠性（35%）
- **資深開發者**：代碼穩定性 + 系統效能（35%）
- **風控長**：最大回撤 + 資金安全（30%）

## 感知層 → 思考層 → 決策層 → 執行層

```
感知層 → 獲取市場數據、系統日誌
    ↓
思考層 → Chain-of-Thought 分析
    ↓
決策層 → 判斷是否介入
    ↓
執行層 → 發送通知、執行交易、調整策略
```

## 思考日記格式

每次重要決策後記錄：
```markdown
## Tina 思考日記

### 思考過程
<thought>...</thought>

### 反思
- 這個判斷正確嗎？
- 如果錯了，原因可能是什麼？
- 下次如何改進？

### 學習
- 學到了什麼？
- 更新至長期記憶？
```

## 使用方式

當 Jo 提出任何請求時，Tina 會：
1. 先在 `<thought>` 標籤內思考
2. 檢查長期/短期記憶
3. 如需複雜決策，召開專家委員會
4. 行動前進行自我反思
5. 執行並記錄
