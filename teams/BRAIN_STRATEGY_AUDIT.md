## 二、現有交易策略邏輯

### 2.1 Leo v6.5 進場邏輯

```
Entry RSI Range: 45-70（TW）/ 40-70（US）
Momentum Filter: 近5日 > 大盤表現
MA20 Deviation: < 20%（避開超漲）
MA20 Above MA60: 需要（多頭確認）
Volume Ratio: > 0.8x
```

**問題診斷：**
- RSI 45-70 進場範圍廣（Good — 不易錯過機會）
- 但缺乏「相對強弱」篩選（Strong Stock篩選）
- MA20偏離20%門檻太寬鬆
- 沒有限制「進場次數」頻率（Cooldown機制有但太短）

### 2.2 停利/停損規則

| 市場 | 停利 | 停損 | 持有天數 |
|:----:|:----:|:----:|:-------:|
| TW | +15% 或 RSI>80+profit>5% | -10% | 最長45天 |
| US | +$300/口 | -$200/口 | 最長45天 |

**問題診斷：**
- TW停利15%合理，但US停利$300對不同價位股票效果差異大
- 沒有所謂「時間停利」（持有多天後，逐步提高停損）
- 持有45天太長（Leo v6.5自己說「縮短持有5→3天」，但實際是45天）

### 2.3 分批進場邏輯

```
初倉：帳戶2%（$2,000/口）
加碼：-5%回調再20%
```

**問題診斷：**
- 分批進場只算「第一次加碼」
- 沒有退場機制（帳面獲利後的管理）
- 缺少「固定股數」規則（每次都是固定金額）

## 三、標準作業流程（SOP）

### 3.1 每日流程（自動）
```
09:00  Leo 每日Paper Trade檢討
  ├─ 抓取即時價格
  ├─ 計算帳面損益
  ├─ 執行停利/停損判斷
  ├─ 平倉 excess positions（>3口）
  └─ 更新 stats + current_price

16:00  Tina 每日DB收盤更新
  ├─ yfinance 增量更新
  └─ finmind 法人資料更新

16:30  Leo v6.5 科技股波段
  ├─ 全市場掃描
  ├─ RSI + 動量 + MA20 評分
  ├─ 計算進場候選
  └─ 寫入 leos_trades.json
```

### 3.2 進場標準流程（人工審核後執行）
```
1. 篩選：RSI 45-70 + 動量 > 大盤 + MA20偏離 < 20% + MA20>MA60
2. 評分：Score > 500（可調整）
3. 計算：股數 = min(100,000 / 進場價, 帳戶2%)
4. 分批：初倉50% + 回調5%加碼50%
5. 寫入：timestamp + entry_price + market + target_price + stop_loss
```

### 3.3 出場標準流程（自動）
```
1. 停利觸發：current >= target_price（TW）或 pnl_abs >= 300（US）
2. 停損觸發：current <= stop_loss（TW）或 pnl_abs <= -200（US）
3. RSI過熱：RSI > 80 + profit > 5% + 持有 > 3天
4. 持倉超限：同一股票 > 3口（平倉最舊的）
5. 最大持有：45天到期（平倉）
```

## 四、改善建議

### 4.1 進場策略改善

| # | 問題 | 建議 | 優先級 |

| M5 | 分批進場只做一次 | 改為「最多3次分批，每次-3%回調」| P2 |

### 4.2 出場策略改善

| # | 問題 | 建議 | 優先級 |
|:-:|:-----|:-----|:------:|
