# 自動化排程蔘考文檔

## OpenClaw Cron 命令

### 基本語法
```
openclaw cron schedule "<crontab>" "<command>"
openclaw cron list
openclaw cron remove <task-id>
openclaw cron test <task-id>
```

### Crontab 格式
```
┌───────────── 分鐘 (0-59)
│ ┌───────────── 小時 (0-23)
│ │ ┌───────────── 日 (1-31)
│ │ │ ┌───────────── 月 (1-12)
│ │ │ │ ┌───────────── 星期 (0-6, 0=週日)
│ │ │ │ │
* * * * *
```

### 常用時間範本

| 時間 | Crontab | 說明 |
|:-----|:--------|:-----|
| 09:00 | `0 9 * * *` | 每天早上 |
| 15:30 | `30 15 * * *` | 每天下午 |
| 16:30 | `30 16 * * *` | 台股收盤 |
| 23:00 | `0 23 * * *` | 美股开盘前 |
| 週六10:00 | `0 10 * * 6` | 每週檢討 |
| 每30分鐘 | `*/30 * * * *` | 頻繁檢查 |
| 每小時 | `0 * * * *` | 每小時 |

## 自動化循環整合

自動化排程可與 `automation-rules` 技能整合：

### 每輪循環完成後自動報告
- 在第10步完成後發送摘要
- 寫入 `memory/automation_progress.md`

### 每日自動化循環檢查
```bash
# 每天 22:00 檢查自動化是否正常運行
openclaw cron schedule "0 22 * * *" "check_automation_loop.py"
```

## 警示級別

| 級別 | 說明 | 觸發條件 | 通知方式 |
|:----:|:-----|:---------|:---------|
| 🔴 緊急 | 重大虧損或月線破位 | RSI>80 + VIF>3 | 立即通知 |
| 🟡 警告 | 價漲量縮或背離 | 成交量<MA50% | 當日通知 |
| 🟢 正常 | 例行監控 | 符合進場條件 | 摘要報告 |

## 故障排除流程

```
問題：排程沒有執行
  │
  ├─ 檢查任務是否存在
  │   └─ openclaw cron list
  │
  ├─ 檢查系統時間
  │   └─ date (Windows: Get-Date)
  │
  ├─ 手動執行腳本
  │   └─ python <script>.py
  │
  └─ 查看日誌
      └─ openclaw logs
```
